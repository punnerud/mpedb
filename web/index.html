This is not a simulation. The COW B+tree, MVCC snapshots, the freelist commit fixpoint, rigid type and CHECK validation, the MPEE join solver and content-hashed plans are all the same Rust code that runs natively — compiled to WebAssembly. Only the kernel is emulated: mmap is a byte array, msync is a no-op, and the writer lock is a flag, because a browser tab is one process with one thread and no durability.

Everything below runs locally in this tab. Nothing is sent anywhere, and there is no server.

Real engine, but no multi-process writers here — why ▸

The browser build is the process-private :memory: path, so mpedb's headline property — several OS processes writing one file, any of them killable mid-commit — is not on display. Two tabs are two WebAssembly instances with separate byte arrays.

Not impossible, just not built. SharedArrayBuffer and Atomics give real futex and CAS primitives; what is missing is owner-death detection, and inventing a third lock construction for a demo is not something we would do quietly (sketch).

Absent for the same reason: durability classes, crash recovery, real files, and the C-API shim — those numbers live in the engine comparison. Everything you can click here is genuine: the SQL semantics, the plans and their hashes, the footprints, the MPEE join reordering, and every refusal.
